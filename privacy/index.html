PUBLIC POLICY

Privacy Policy

Effective July 20, 2026. This policy explains what UpTier handles, what stays local, and when optional services process information outside your device.

Overview

UpTier is operated by Dylan Hunter in Texas, United States. It is a local-first Windows application for portfolio, dividend, calendar, budget, and financial-freedom planning. Core local features do not require an UpTier cloud account.

The Windows app does not currently include advertising, behavioral tracking, third-party analytics, or automatic crash-report uploads. Questions and privacy requests may be sent to uptier.support@gmail.com.

Information stored locally

Depending on what you enter or import, the encrypted local Profile Vault can contain portfolio positions and history; CSV or supported text-based PDF content; income, bills, debts, budgets, goals, calendar entries and planning assumptions; profile names and settings; and optional market-data credentials.

Supported CSV and PDF files are parsed locally. UpTier does not upload the source statement or brokerage file merely because it was imported. Windows protected storage protects profile keys, provider credentials, and protected account sessions for the current operating-system user.

Optional cloud identity

The private connected-portfolio Trial uses Google Identity Platform/Firebase Authentication for verified email, password authentication, and SMS multi-factor authentication. Google processes the account email, password-verification material, mobile number, MFA enrollment, and related anti-abuse data under Google's terms and privacy practices.

The UpTier backend receives a signed identity proof containing a Firebase account identifier, verified email, and confirmation that phone MFA was completed. UpTier does not receive the readable Google/Firebase password or SMS code. It stores the account email, provider identifier, hashed sessions, entitlement state, encrypted connected-provider state, and narrowly scoped security events.

Google Cloud Run hosts the private backend. A private Google Cloud Storage object retains an encrypted recovery snapshot. Cloud infrastructure logs can contain request time, network address, device or browser metadata, response status, and security events.

Connected portfolio and Plaid

Before the first connection under the current disclosure version, UpTier identifies the data categories, purposes, recipients, retention, and withdrawal controls and requires affirmative consent.

With that authorization, Plaid and the UpTier backend can process institution and investment-account names and identifiers; supported balances, holdings, quantities, prices, and cost information; security details; and supported investment-transaction and dividend history. Brokerage authentication occurs in Plaid's hosted experience. UpTier does not intentionally collect or store brokerage login credentials.

Encrypted Plaid access state remains on the backend so an authorized connection can refresh. The desktop app receives a bounded normalized portfolio response and saves imported information in the encrypted local Profile Vault. Plaid also processes information under Plaid's End User Privacy Policy.

Sandbox and private Trial under validation. The connection remains an invitation-only real-data Trial and is not generally available.

Other optional providers

If Alpha Vantage is configured, UpTier sends requested ticker symbols and the user's Alpha Vantage API key over HTTPS to obtain supported market and dividend information. Microsoft can process Store identity, acquisition, license, payment, and diagnostic information when a Store version or future Store purchase is used. UpTier does not currently operate a public paid subscription or receive complete card numbers.

Use and disclosure

UpTier uses information to provide requested calculations, authenticate an optional account, maintain security, connect or refresh an authorized portfolio, reconcile holdings and dividend history, provide support, prevent abuse, and comply with law.

UpTier does not sell personal information and does not use connected financial data for targeted advertising or behavioral profiling. Information is disclosed only to providers needed for a feature you deliberately enable, to protect users and the service, or when legally required.

Retention and deletion

  • Local information remains until you edit or delete it or remove the applicable UpTier application-data folder.
  • Desktop cloud sessions expire after seven days. Browser sign-in and deletion requests expire after ten minutes.
  • Cloud account and encrypted Plaid state remain until you disconnect, withdraw consent, or delete the cloud account.
  • Disconnecting removes connected Plaid Items and server access tokens while retaining records already imported locally.
  • Verified cloud-account deletion removes connected Plaid Items, UpTier account and session records, encrypted server state, and the Google/Firebase identity record. Local records remain until deleted locally.
  • Operational audit events and webhook replay hashes have a 180-day target retention and are automatically purged at service startup and during request processing after crossing that target. Providers can retain limited records when independently required for security, fraud prevention, accounting, or law.

See the public Data Deletion instructions for exact controls and support options.

Your choices and requests

You can decline a connection, disconnect Plaid, withdraw connected-data consent, sign out, delete the cloud account, clear a credential, export supported local information, or delete local data. You may request access, correction, or deletion of information controlled by UpTier, or appeal a privacy-request decision, by emailing private support. UpTier will verify the requester using the minimum information reasonably necessary.

Security and incidents

UpTier uses encryption in transit, encrypted cloud snapshots, hashed or encrypted authentication records, bounded requests, least-privilege service identities, and MFA. No security method guarantees absolute protection. UpTier will follow applicable notification requirements if an incident requires notice.

Age eligibility

Cloud accounts and connected financial-data features are intended for adults age 18 or older. UpTier is not directed to children under 13 and does not knowingly offer cloud accounts to them.

Changes and contact

Material changes will be posted here with a new effective date. When a change materially affects connected financial-data processing, UpTier will require acceptance of a new consent version before another connection or synchronization. Never send statements, credentials, tokens, account numbers, or financial attachments by ordinary email.
