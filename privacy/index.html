PUBLIC POLICY

Privacy Policy

Effective July 18, 2026. This policy explains what UpTier handles, what stays local, and when optional services can send information outside your device.

Overview

UpTier is a local-first Windows application for portfolio, dividend, calendar, budget, and financial-freedom planning. UpTier does not require an UpTier account for its local Windows features. The current Windows app does not include advertising, behavioral tracking, third-party analytics, or automatic crash-report uploads.

Information UpTier handles

Depending on the features you use, UpTier can handle:

  • portfolio positions, prices, invested amounts, cost basis, dividend history, and investment-account labels;
  • broker and bank data you import from supported CSV or text-based PDF files;
  • income, recurring bills, debts, pay schedules, budgets, goals, and planning assumptions;
  • local profile names and application settings;
  • optional market-data credentials and optional connected-portfolio responses; and
  • for a private connected-portfolio Trial, an UpTier account email plus encrypted or hashed authentication, session, recovery, and security-audit records.

UpTier does not intentionally collect brokerage passwords or bank login credentials. UpTier account passwords are processed only to verify the account and are stored as salted password hashes, not readable password text.

Local storage and security

Financial profiles are stored on the device in an encrypted Profile Vault. Each profile has its own encryption key, and Windows protected storage protects that key for the current operating-system user. Provider credentials are stored separately using Windows protected storage and are not displayed again after saving.

Supported CSV and PDF imports are parsed locally by the Windows application. UpTier does not upload a source statement or broker file as part of the local import process. Development builds use a separate local data directory and do not read or modify an installed UpTier profile.

Optional external services

UpTier sends information outside the device only when you deliberately configure and use an optional external service.

Alpha Vantage

UpTier sends requested ticker symbols and your Alpha Vantage API key over HTTPS to retrieve supported market and dividend information. Alpha Vantage processes those requests under its own terms and privacy policy.

Plaid and a connected-portfolio backend

When you choose to connect or refresh a portfolio, the Windows app communicates over HTTPS with a configured secure UpTier backend. Brokerage authentication occurs through Plaid's hosted experience. Brokerage login credentials and Plaid secrets are not stored in the desktop application.

With your authorization, the backend can process institution and investment-account details, supported balances and holdings, security details, and supported investment-transaction or income history. Encrypted Plaid connection state remains on the private backend so an authorized connection can be refreshed. UpTier uses synchronized information to reconcile a portfolio and supported dividend history in your local Profile Vault. It is not used for advertising.

The Plaid connection is being validated in Sandbox and in a limited, invitation-only real-data Trial. It is not generally available. No broader release will occur until Plaid review, production security requirements, and release testing are complete.

Plaid's processing is also governed by Plaid's End User Privacy Policy.

Sharing and selling

UpTier does not sell personal information. The current Windows app does not send local financial data to the UpTier developer for advertising or analytics. Information is disclosed to an optional provider only as needed for the feature you deliberately configure and use, when necessary to protect the service, or when required by law.

Retention and deletion

Local information remains on the device until you edit or remove it, clear a saved credential, or remove the corresponding application-data files. The direct Windows installer can preserve the encrypted Profile Vault during uninstall to reduce accidental data loss.

For a private connected-portfolio Trial, account and encrypted connection state remain on the backend until the connection or Trial account is deleted. Desktop sessions expire after seven days and short-lived authenticator enrollment records expire after ten minutes. Disconnecting removes the corresponding Plaid Item and backend access token. Deleting the Trial account removes its sessions, encrypted Plaid state, and identifying account record; limited de-identified integrity or security-event hashes may remain where needed to prevent replay or investigate abuse.

Instructions for removing all local UpTier information and deleting connected-service data are available on the Data Deletion page. Back up anything you need before deleting local files.

Your choices

You can use the core local features without Plaid or a market-data provider. You choose whether to configure a provider, connect an account, refresh connected data, import a file, disconnect a brokerage, delete a Trial account and its server data, or delete local data.

Children

UpTier is not directed to children under 13 and is not designed to collect children's personal information.

Changes and contact

Material changes will be posted on this page with a new effective date. For privacy questions or requests, email uptier.support@gmail.com or follow the Support instructions. Never send bank statements, brokerage exports, credentials, tokens, account numbers, or other sensitive financial information unless UpTier first provides an appropriate secure transfer method.
