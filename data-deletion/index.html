Data Deletion

Use these instructions to permanently remove local UpTier information or request deletion of data associated with an optional connected service.

Delete local UpTier data

Deletion is permanent. Export or back up anything you need before continuing.

  1. Close UpTier completely.
  2. Uninstall the application if you no longer intend to use it.
  3. In Windows File Explorer, open %APPDATA%.
  4. Delete the UpTier folder to remove installed-app profiles, encrypted vaults, local settings, saved provider credentials, caches, and recovery generations.
  5. If you used Visual Studio or an unpackaged development build, also delete the separate UpTier-Dev folder if present.

Uninstalling the direct Windows installer by itself can preserve local data to prevent accidental loss. Deleting the application-data folder is the step that removes it.

Delete connected-service data

The Plaid integration is currently limited to Sandbox and invitation-only real-data Trial validation. A signed-in Trial user can select Disconnect Brokerage in UpTier to remove every connected Plaid Item and backend access token while retaining local imported records. The user can select Delete Trial Account and Server Data, confirm the account password, and remove the account's sessions, encrypted Plaid state, and identifying account record.

If the in-app deletion controls are unavailable, email uptier.support@gmail.com with the subject UpTier data deletion request. Do not include passwords, bank statements, account numbers, Plaid credentials, access tokens, or other financial records.

Request connected-data deletion

After identity and connection ownership are verified through a private method, UpTier support will remove or revoke data the developer controls unless retention is required by law, security, or fraud-prevention obligations. Provider-held data may also be governed by the provider's own deletion process and retention requirements.

What to include initially

In the first email, include only:

  • that the request concerns an UpTier connected portfolio, Sandbox test, or private Trial;
  • the approximate date the connection was created; and
  • a request for private verification instructions.

Do not include your institution, account number, Plaid identifiers, access token, statements, screenshots, exported financial data, passwords, or MFA credentials. UpTier support will provide the minimum verification instructions needed for the request.

Questions

For questions about local deletion, connected services, or the handling of a request, follow the Support instructions or review the Privacy Policy.
