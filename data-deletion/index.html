Data Deletion

Use these instructions to permanently remove local UpTier information or verified cloud-account and connected-service data.

Delete local UpTier data

Deletion is permanent. Export or back up anything you need before continuing.

  1. Close UpTier completely.
  2. Uninstall the application if you no longer intend to use it.
  3. In Windows File Explorer, open %APPDATA%.
  4. Delete the UpTier folder to remove installed-app profiles, encrypted vaults, local settings, saved provider credentials, caches, and recovery generations.
  5. If you used Visual Studio or an unpackaged build, also delete UpTier-Dev if present.

The direct installer can preserve local data during uninstall to prevent accidental loss. Removing the application-data folder is the permanent local deletion step.

Disconnect a connected portfolio

In Settings → Connected Portfolio, select Disconnect Brokerage to remove connected Plaid Items and server-side access tokens while retaining financial records already imported into the encrypted local profile. Selecting Withdraw Consent and Disconnect also clears UpTier's saved connected-data consent.

Delete a Google/Firebase cloud account

  1. Sign in to UpTier and open Settings → Connected Portfolio.
  2. Select Delete Cloud Account and Server Data.
  3. UpTier opens the secure browser portal. Re-enter the account email and password and complete SMS MFA.
  4. After verification, UpTier removes connected Plaid Items, backend access state, sessions, account records, and the Google/Firebase identity record.
  5. Return to UpTier. The desktop app confirms deletion and clears its protected session.

This cloud action does not delete portfolio, dividend, Budget, or Calendar records already imported locally. Use the local instructions above when those records must also be removed.

Delete a local-TOTP Trial account

Early private-Trial accounts that use an in-app authenticator code instead of Google/Firebase can select Delete Trial Account and Server Data and confirm the account password in UpTier. The backend removes its Plaid Items, sessions, encrypted state, and account record.

If a control is unavailable

Email uptier.support@gmail.com with the subject UpTier data deletion request. Initially state only that the request concerns an UpTier cloud account or connected portfolio and ask for private verification instructions. Do not include passwords, statements, account numbers, Plaid identifiers, tokens, screenshots, or financial attachments.

Request connected-data deletion

After ownership is verified through a private method, UpTier will remove data it controls unless limited retention is independently required for security, fraud prevention, accounting, or law. Provider-controlled information may require the provider's own deletion process.
